  <action>
Create `backend/src/controllers/recipeController.js` with these methods:

1. **createRecipe(req, res)**:
   - Extract from req.body: title, description, category, difficulty, cooking_time, servings, instructions, ingredients (JSON string)
   - Validate required fields, return 400 if missing
   - Check req.file exists, return 400 with message `'Recipe image is required'` if not
   - Build image_url: `/uploads/recipes/${req.file.filename}`
   - Parse ingredients: `JSON.parse(ingredients)`
   - Create recipe using Recipe.create() with user_id from req.user.id
   - For each ingredient:
     - Use Ingredient.findOrCreate(ing.name) to get/create ingredient
     - Use RecipeIngredient.create() with recipe_id, ingredient_id, quantity, unit
   - Return 201 with { success: true, recipe }
   - Wrap in try/catch, return 500 on error with cleanup: `fs.unlinkSync(req.file.path)` if file exists

2. **getRecipeById(req, res)**:
   - Get recipe using Recipe.findById(req.params.id)
   - Return 404 if not found with message `'Recipe not found'`
   - Get ingredients using RecipeIngredient.findByRecipeId(recipe.id)
   - Return 200 with { success: true, recipe: { ...recipe, ingredients } }

3. **getAllRecipes(req, res)**:
   - Extract query params: category, difficulty, minRating, search, sortBy, page (default 1), limit (default 12)
   - Calculate offset: `(page - 1) * limit`
   - Call Recipe.findAll() with filters
   - Calculate total count for pagination
   - Return 200 with { success: true, recipes, pagination: { page, limit, total, totalPages } }

4. **getMyRecipes(req, res)**:
   - Get recipes using Recipe.findByUserId(req.user.id)
   - Return 200 with { success: true, recipes }

5. **updateRecipe(req, res)**:
   - Get existing recipe using Recipe.findById(req.params.id)
   - Return 404 if not found
   - Check ownership: `recipe.user_id !== req.user.id` return 403 with message `'Not authorized to edit this recipe'`
   - Build updates object from req.body (only include non-undefined fields)
   - If req.file exists, update image_url and delete old image file
   - If ingredients provided, delete existing via RecipeIngredient.deleteByRecipeId() and recreate
   - Call Recipe.update(id, updates)
   - Return 200 with { success: true, recipe }

6. **deleteRecipe(req, res)**:
   - Get recipe using Recipe.findById(req.params.id)
   - Return 404 if not found
   - Check ownership, return 403 if not owner
   - Delete image file from filesystem if exists
   - Call Recipe.delete(id) - cascade deletes handle related records
   - Return 200 with { success: true, message: 'Recipe deleted successfully' }

Export all methods: `module.exports = { createRecipe, getRecipeById, getAllRecipes, getMyRecipes, updateRecipe, deleteRecipe };`

1. Create `backend/src/routes/recipeRoutes.js`:

```javascript
const express = require('express');
const router = express.Router();
const recipeController = require('../controllers/recipeController');
const { verifyToken } = require('../middleware/authMiddleware');
const upload = require('../middleware/uploadMiddleware');

// Public routes (no auth required)
router.get('/', recipeController.getAllRecipes);
router.get('/:id', recipeController.getRecipeById);

// Protected routes (auth required)
router.post('/', verifyToken, upload.single('image'), recipeController.createRecipe);
router.put('/:id', verifyToken, upload.single('image'), recipeController.updateRecipe);
router.delete('/:id', verifyToken, recipeController.deleteRecipe);
router.get('/user/me', verifyToken, recipeController.getMyRecipes);

module.exports = router;
```

Note: Place `/user/me` route BEFORE `/:id` to prevent 'me' being parsed as an ID, OR ensure proper route ordering.
